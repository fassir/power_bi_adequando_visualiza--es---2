

# Desafio: Atualizando Relatório Financeiro com Foco na Experiência do Usuário

- Botões de navegação e próximos passos

## Objetivo do desafio

Modificar o relatório (o relatório criativo que foi criado anteriormente) com foco na experiência do usuário (UX). Acompanhe os vídeos do curso para entender as decisões tomadas durante o processo e, ao refatorar o relatório, leve em consideração os seguintes pontos:

- Posicionamento
- Contraste
- Proporção áurea
- Segmentação dos dados

> Observação: Esses pontos são guias de design, não regras rígidas. Use-os para orientar suas escolhas, mas saiba quando quebrar as regras para aumentar a criatividade e melhorar a comunicação dos dados.

## Requisitos / Instruções

1. Inserir botões de navegação entre páginas (botões "Go to Page" / "Page").
2. Modificar a segunda página de forma similar ao demonstrado no desafio para a primeira página.
3. Ajustar os botões de navegação para destacar ações de foco e seleção (por exemplo: estado normal, hover/foco, selecionado).
4. Criar menus de navegação em cada página, se necessário, para facilitar a movimentação do usuário.
5. Modificar e melhorar um gráfico de área conforme demonstrado (legibilidade de eixos, cores e preenchimentos).
6. Construir uma matriz de descrição de vendas (tabela/matriz que explique categorias, métricas e comparações relevantes).
7. Ajustar estilização dos visuais e da página (tipografia, cores, alinhamentos, espaçamento e proporção visual).
8. Garantir que o relatório tenha 3 páginas funcionais e coerentes entre si.
9. O estilo dos botões é livre — use a identidade visual que achar adequada para a leitura e o contraste.

## Critérios de avaliação

- Clareza e legibilidade: títulos, labels e legendas devem ser fáceis de ler e interpretar.
- Navegabilidade: botões e menus devem facilitar a movimentação entre as páginas.
- Consistência visual: paleta de cores, tipografia e espaçamento coerentes.
- Segmentação e insights: capacidade de filtrar e segmentar dados para responder perguntas de negócios.
- Criatividade e boas justificativas: bom uso das regras de design e justificativas quando optar por quebrá-las.

## Entregáveis

- Arquivo do relatório atualizado (`.pbix`) ou export em PDF com páginas navegáveis.
- 2–3 screenshots mostrando cada página e os estados dos botões (normal, foco, selecionado).
- Breve documento ou notas (1 página) explicando as principais mudanças e motivos de design.

## Como entregar

- Commit no repositório com o arquivo `report.pbix` (ou `report.zip`) na pasta `deliverables/` e adicione screenshots em `deliverables/screenshots/`.
- Alternativamente, compacte os arquivos em um `.zip` e envie conforme combinado (e-mail / plataforma do curso / pull request).
